# Space Overview

## 用途

管理第一家线下店中与 Session 01 MVP 相关的空间需求。

## 当前原则

- 空间不是起点，空间服务 Session 01 的 User Journey 和 Experience Detail。
- Journey Room 是当前核心。
- Arrival、Landing Room、Cafe 只做必要衔接。
- 装修与采购只记录可落地、可测试的内容。

## 日常开发方式

先看完整动线，再看每个空间承担什么体验。

空间配置必须在 User Journey 和 Experience Detail 明确后再定义。

优先回答:

- 用户怎么走？
- 用户在哪里停留？
- 每个空间让用户感觉到什么？
- 有没有多余空间或多余改造？
- 什么配置是测试必须的？

## 新增空间 Challenger 五问

1. 它解决了什么问题？
2. 如果没有它，Session 还能成立吗？
3. 有没有更简单的方法？
4. 是否已经有其他产品承担了这个功能？
5. 如果必须删掉一个产品，它会不会是最先被删掉的那个？

如果不能明确通过，先进入 [Parking Lot](../Parking%20Lot（以后再做）/ParkingLot.md)。

## 空间模块

| 区域 | 当前作用 | 状态 |
| --- | --- | --- |
| Arrival | 到店、进入、情绪切换 | ☐ |
| Journey Room | Session 01 核心体验 | ☐ |
| Landing Room | Session 后沉淀与缓冲 | ☐ |
| Cafe | 等候、陪伴、测试后交流 | ☐ |
| 装修 & 采购 | 真实落地执行 | ☐ |
