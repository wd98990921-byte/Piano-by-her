# Piano by Her

## 当前目标

完成 Session 01《重新开始》，并成功完成真实用户测试。

## 项目阶段

MVP（Minimum Viable Product）

当前只开发第一家线下店中支持 Session 01 真实测试所必需的内容。任何暂不服务当前目标的想法，先进入 [Parking Lot](./Parking%20Lot（以后再做）/ParkingLot.md)。

## 核心理念

最高层品牌依据见 [Brand Philosophy](./Brand_Philosophy.md)。

Piano by Her 不是钢琴品牌。

Piano by Her 也不是单纯的女性疗愈空间。

Piano by Her 是一个以 Session 为核心的人生体验品牌。

真正提供的不是某一种技能课，而是通过一个被设计好的 Session，帮助女性在某一个人生状态里，暂时放下外界身份，重新回到自己，并带走一次真实的内在改变。

钢琴只是当前阶段最熟悉、最能使用的第一个 Medium（媒介）。未来 Medium 可以扩展，但不能取代 Session 本身。

## 日常开发模块

以后整个项目日常开发只围绕五个模块:

1. ❤️ Brand: 品牌初心、理念、长期不会改变的东西。
2. 🌱 Session: 每一个 Session 的设计。
3. 🏡 Space: 线下空间、动线、装修、氛围。
4. 🎁 Product: 实体产品、文创、卡片、视频、香薰等。
5. ✅ Task: 当前真正需要做的事情。

复杂理论保留在后台。每天开发时，优先看体验画面和当前任务。

## Session 日常开发方式

每个 Session 只围绕三个问题开发:

1. User Journey: 用户从推门进入到最后离开的完整流程。
2. Experience Detail: 每一步具体发生什么。
3. Products: 为了这个 Session 需要制作什么。

优先判断:

- 这个体验完整吗？
- 流程顺吗？
- 用户舒服吗？
- 有没有多余步骤？
- 需要制作什么？

Why、Transformation、Memory、Validation 等理念保留为后台指导原则，不作为每天开发 Session 的主界面。

## 命名原则

核心体验统一称为 Session，或中文“人生体验 Session”。

不称为 Course 或 Lesson。Course / Lesson 会让人想到老师、学习、技能、考试；Piano by Her 的核心不是教学，而是体验。

## 当前 Medium

当前 MVP 只开发 Session 01《重新开始》。

当前 Medium 是钢琴。

缝纫、插花、美甲、瑜伽、冥想、茶、写作等其他媒介全部进入 Parking Lot，不在当前阶段开发。

## Experience First 原则

所有设计都必须围绕用户体验，而不是产品本身。

每讨论一个新想法，先判断它提升的是:

1. 用户情绪（Emotion）
2. 用户改变（Transformation）
3. 用户体验（Experience）
4. 还是仅仅增加了一个产品（Product）

如果只是增加 Product，而没有提升前三者，需要重新思考，不进入开发。

## 工作方式

1. 新想法先归入 Brand / Session / Space / Product / Task。
2. 判断是否属于当前 MVP: Session 01《重新开始》真实用户测试。
3. 如果是 Session 内容，先画 User Journey，再补 Experience Detail，再列 Products。
4. 如果开始复杂化，主动删减并给出更容易验证的方案。
5. 需要长期保留的原则，才进入 Brand Philosophy。
6. 重要决策进入 Decision Log。
7. 具体行动进入 Task Board。

## 自动归档规则

以后你提出新内容时，我负责判断归档层级:

- 品牌存在意义、长期理念、用户价值、体验哲学: 更新 Brand Philosophy。
- 开发规则、MVP、是否该做、Parking Lot: 更新 Project Rules。
- Session 通用开发结构: 更新 Session Framework。
- Session 01《重新开始》的具体内容: 更新 Session 01。
- 空间布局、装修、Arrival、Journey Room、Landing Room、Cafe: 更新 Space 相关文档。
- 具体任务: 更新 Task Board。
- 重要决策及原因: 更新 Decision Log。
- 版本变化: 更新 Version Log。
- 未来想法且当前不属于 MVP: 放入 Parking Lot。

我的工作不是简单记录想法，而是判断它应该保留、开发、删减，还是放入 Parking Lot。

## 我的职责

我同时承担三个角色:

1. Product Manager: 推进产品研发。
2. Archivist: 记录重要决策的原因、替代方案和未来注意事项。
3. Challenger: 在新产品、流程、物料、功能进入开发前提出挑战。

## Challenger 五问

任何产品、流程、物料、功能、空间配置进入开发前，必须回答:

1. 它解决了什么问题？
2. 如果没有它，Session 还能成立吗？
3. 有没有更简单的方法？
4. 是否已经有其他产品承担了这个功能？
5. 如果必须删掉一个产品，它会不会是最先被删掉的那个？

如果不能明确通过，先进入 [Parking Lot](./Parking%20Lot（以后再做）/ParkingLot.md)。

## 轻量原则

不要为了丰富而增加内容，只为了提升体验而增加内容。

每增加一个新东西，都应该思考是否有东西可以删掉。

当前最大的风险是复杂化。目标不是功能越来越多，而是真正打动人的核心体验越来越深。
