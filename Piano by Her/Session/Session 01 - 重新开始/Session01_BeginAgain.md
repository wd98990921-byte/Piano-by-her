# Session 01 - 重新开始

## 定位

当前唯一核心 Session。

Session 名称: 重新开始 | Restart。

当前正式曲目: Lost in Island。

Session 01《重新开始》不是一节钢琴课，也不是一次单纯的演奏体验。它是一段关于“她想重新开始”的 Life Experience。

真正需要被设计的不是钢琴课，而是她从“我已经很久没有只做自己了”走向“我重新相信自己可以开始”的 Transformation。

当前 Medium 是钢琴。钢琴不是 Session 主体，只是当前阶段帮助用户完成 Transformation 的第一个媒介。

## MVP 目标

让真实用户完成一次关于“重新开始”的线下 Journey，并验证:

- 用户是否能理解这次体验回应的人生命题。
- 用户是否在体验中发生可感知的内心变化。
- 用户是否愿意完整参与 Arrival → Journey → Landing → Cafe。
- 钢琴作为媒介，是否帮助 Transformation 发生。
- 用户是否觉得今天值得，并愿意再次预约或推荐朋友。

## 当前推进规则

日常开发只围绕三个问题:

1. User Journey: 用户从推门进入，到最后离开的全过程。
2. Experience Detail: 每一步具体发生什么。
3. Products: 为了这个 Session 需要制作什么。

Why、Transformation、Memory、Validation 作为后台指导原则保留，不作为每天开发的主界面。

当前 Session 01 的 Medium 已确定为钢琴。Medium 是支持系统，不改变本轮主开发顺序。

## 体验周期

一首歌建议通过 1 个月、4 次体验完整完成。

4 次体验不是强制要求:

- 客人可以只体验 1 次。
- 客人可以体验 2 次后切换其他 Session。
- 客人可以随时选择新的 Session。

只有完整完成该 Session 的客人，才会拍摄专属成品弹奏视频 / Journey Film（美丽博主版）。

## 正式交付物

每位体验《重新开始》的客人，可获得:

- Session Card。
- 临别赠卡。
- Journey Photo。
- 完整 Lesson Video。

只有完整完成该 Session 的客人，再获得:

- 专属 Journey Film / 成品弹奏视频。

目前不把“客人当堂弹奏手部录像”作为标准交付物。一次体验后客人未必能演奏得足够完整，照片更适合作为单次体验纪念。

## ① Why: 为什么会有这个 Session

重新开始，并不是帮助用户解决人生问题。

Session 01 提供的是一个空间，让她暂时放下所有身份，重新连接自己。

它不是逃离现实，而是在体验结束以后，带着一点勇气，重新走回现实。

| 状态 | 内容 |
| --- | --- |
| ☑ | 回应“我已经很久没有只做自己了”的人生状态 |
| ☑ | 不承诺解决人生问题，只提供一次完整的人生体验 |
| ☑ | 与 SPA、按摩、咖啡区分: 它们更多解决身体放松，Session 01 关注内在重新连接 |
| ◐ | 验证用户是否能在体验后带着一点勇气重新走回现实 |

## ② Who: 她是谁

当前目标用户为 22-50 岁女性。

| 状态 | 内容 |
| --- | --- |
| ☑ | 工作压力很大 |
| ☑ | 已经拥有稳定收入，却越来越找不到快乐 |
| ☑ | 长期照顾别人，却很少照顾自己 |
| ☑ | 很久没有认真体验自己的生活 |
| ☑ | 电量越来越低 |
| ☑ | 想暂停一下人生 |
| ☐ | 她和钢琴 / 音乐的关系 |
| ☐ | 她不是谁: 本次 MVP 暂不服务的人群 |

## ③ Before: 来到这里之前

Before 不只记录“焦虑、疲惫、压力”，而要记录更真实的人生状态。

| 状态 | 内容 |
| --- | --- |
| ☑ | 我已经很久没有只做自己了 |
| ☑ | 我每天照顾别人，却很少照顾自己 |
| ☑ | 我忘记自己喜欢什么 |
| ☑ | 我一直在努力，但不知道为什么努力 |
| ◐ | 她今天为什么会预约这个 Session |
| ◐ | 她可能的犹豫、紧张或抗拒 |

## ④ Transformation: 改变

这是 Session 01 最重要的一层。

Session 不负责改变人生。

Session 负责改变她看待人生的方式。

| 状态 | 内容 |
| --- | --- |
| ☑ | 她刚来时: 我已经很久没有只做自己了 |
| ☑ | 她在体验中: 暂时放下外界身份，重新感受自己 |
| ☑ | 她离开时: 不是已经重新开始，而是重新相信自己可以开始 |
| ☑ | 核心改变: 从“我不知道怎么继续”到“我也许可以重新相信自己” |
| ☐ | 这个改变如何被测试观察或访谈验证 |

## ⑤ After: 离开的时候

After 不定义“学会一首歌”，也不定义钢琴水平。

真正希望发生的是: 她相信自己可以重新开始。

| 状态 | 内容 |
| --- | --- |
| ☑ | 重新相信自己的感觉 |
| ☑ | 一点带回现实的勇气 |
| ☑ | 一种更轻、更松、更完整的自己 |
| ◐ | 一段音乐或声音记忆 |
| ◐ | 一张照片或体验痕迹 |

## ⑥ Journey: 体验流程

只有在 Why、Who、Before、Transformation、After 确定后，才进入完整流程设计。

Journey 不只是流程，而是为什么她需要经历这一切。

以后 Journey 中每一步都必须回答:

1. 为什么存在？
2. 解决什么问题？
3. 如果删掉，体验会失去什么？

| 顺序 | 步骤 | Why | 当前状态 |
| --- | --- | --- | --- |
| 1 | Arrival | 让她从日常和外界身份中进入 Session | ◐ |
| 2 | 洗手 | 请把今天留在水里；这是进入体验的仪式 | ◐ |
| 3 | 护手霜 | 今天，你的双手不是为了工作，只是为了感受 | ◐ |
| 4 | 主题卡片 | 降低答题感，让情绪快速进入 | ◐ |
| 5 | 点香 | 通过气味提示“世界留在门外” | ☐ |
| 6 | 钢琴体验 | 作为 Medium 帮助她完成 Transformation | ☐ |
| 7 | 拍照 | 作为可能的 Memory 载体，待测试验证必要性 | ☐ |
| 8 | 休息 | 不急着回到现实，给体验留出回落时间 | ☐ |
| 9 | Landing Room | 重新进入现实之前的缓冲空间 | ◐ |
| 10 | Cafe | 提供停留的自由，不规定社交或独处 | ◐ |

## Arrival 升级

| 元素 | Why | 当前状态 |
| --- | --- | --- |
| 欢迎回来 | 让她感觉这里不是考场，也不是任务场，而是可以暂时回到自己的地方 | ◐ |
| 请把世界留在门外 | 明确 Session 与外界身份、压力和责任的边界 | ◐ |
| 洗手 | 不是为了卫生解释，而是“请把今天留在水里” | ◐ |
| 护手霜 | 不是保养产品，而是“今天，你的双手不是为了工作，只是为了感受” | ◐ |

## 卡片升级

不要直接问“你想重新开始什么”。这会产生压力。

改为:

- 今天，请送给自己一句话。
- 请选择一个今天属于你的词。

候选词:

- 重新
- 勇气
- 自由
- 相信
- 温柔
- 原谅
- 开始

目标是降低答题感，提高情绪进入速度。

## Piano Moment

钢琴不是 Session 主体。

钢琴只是 Journey 中帮助用户完成 Transformation 的一个媒介，不是整个体验的中心。

## 钢琴教学方式

钢琴环节正式采用老师现场开始教学，再把空间还给客人的方式。

老师现场出现并进行教学:

- 帮助客人找到手位。
- 示范旋律。
- 带客人完成最初的学习。
- 不采用全程录播教学。

客人独立练习:

- 老师离开空间。
- 客人观看 Lesson Video，自行回忆和练习。
- 有需要时按铃，老师再进入短暂协助。
- 协助完成后老师再次离开。

核心原则:

老师先带她开始，再把空间还给她；需要时，再回来。

| 状态 | 内容 |
| --- | --- |
| ☑ | 老师先现场出现并教学 |
| ☑ | 客人随后独立练习 |
| ☑ | Lesson Video 作为记忆辅助 |
| ☑ | 按铃呼叫老师短暂协助 |
| ☐ | 4次体验各自的教学进度安排 |

## Lesson Video

Lesson Video 取代传统五线谱，承担客人的“电子曲谱 / 记忆辅助”功能。

客人不会看到五线谱。

五线谱只供老师内部教学使用。

Lesson Video 内容为完整标准示范视频:

- 展示双手与琴键。
- 保留完整演奏。
- 不做单手版、双手版、分段教学版。
- 不增加逐课进度说明。
- 不在结尾增加“今天完成了什么”“下次学什么”等个性化内容。

统一片头约 2-3 秒:

```text
Piano by Her
Session 01
重新开始
Restart
```

不需要结尾动画、二维码、营销信息或个性化剪辑。

每位第一次体验《重新开始》的客人，课程结束后都会收到同一支完整 Lesson Video。

后续再次体验时继续参考同一支视频，无需重复制作或更新。

Lesson Video 的作用:

- 回忆课上学过的内容。
- 回家想弹时可以参考。
- 下次到店前快速恢复记忆。

## 取消方案

以下方案正式取消，不再继续开发:

- 客人用五线谱。
- 数字谱。
- Journey Guide。
- 琴键定位图。
- 左右手轨道图。
- 动态键盘。
- 琴键贴纸。
- 单手或双手拆分教学视频。
- 每节课制作不同版本的视频。

原因:

客人不需要在离店后承担练习任务，视频比纸质谱更适合承担记忆与回顾功能。

## Landing Room 升级

Landing Room 不是结束，也不是休息室。

它的真正作用是: 重新进入现实之前的缓冲空间。

| 空间 | 定位 |
| --- | --- |
| Journey Room | 人生暂停 |
| Landing Room | 重新准备面对世界 |

## Cafe 升级

Cafe 不定义为社交空间。

它提供的是停留的自由。

她可以:

- 聊天。
- 发呆。
- 喝咖啡。
- 看书。
- 什么都不做。

都可以。不规定。

## ⑦ Memory: 记忆

| 状态 | 内容 |
| --- | --- |
| ☐ | 三个月以后，她为什么还会想起今天 |
| ☐ | 什么东西会留下记忆 |
| ☐ | 什么东西会让她想回来 |
| ☐ | 她可能在什么日常瞬间重新想起这次体验 |

## ⑧ Product: 产品与物料

Product 不是起点。

产品只是工具，Transformation 才是产品。

当前产品与物料必须从 Journey 和 Memory 自然生长出来。

## ⑨ Space: 空间

Space 不是先被设计出来的场景，而是承载 Journey 的条件。

空间必须服务 Transformation，而不是制造装饰感。

## Validation: 验证

不使用“学会钢琴”作为成功标准。

| 状态 | 体验指标 |
| --- | --- |
| ☐ | 用户是否真正放松 |
| ☐ | 用户是否能说出自己今天经历了什么 |
| ☐ | 用户是否觉得今天值得 |
| ☐ | 用户是否愿意再次预约 |
| ☐ | 用户是否愿意推荐朋友 |
| ☐ | 用户是否主动分享体验 |
| ☐ | 用户是否在离开后仍然想起这次体验 |

## 新增内容挑战规则

任何产品、流程、物料、空间配置进入 Session 01 前，必须回答:

1. 它解决了什么问题？
2. 如果没有它，Session 还能成立吗？
3. 有没有更简单的方法？
4. 是否已经有其他产品承担了这个功能？
5. 如果必须删掉一个产品，它会不会是最先被删掉的那个？

如果不能明确通过，先放入 [Parking Lot](../../Parking%20Lot（以后再做）/ParkingLot.md)。

## 品牌原则

不要先思考产品是什么。

先思考用户离开的时候，变成了什么样的人。

产品只是工具。

Transformation 才是产品。
